**Create `src/app/api/books/route.ts`** -- POST endpoint for creating books:

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { z } from 'zod'
import { BookStatusEnum } from '@/lib/books/schema'
import { writeBook } from '@/lib/books/library-service'

export const dynamic = 'force-dynamic'

const CreateBookSchema = z.object({
  title: z.string().min(1),
  author: z.string().min(1),
  isbn: z.string().optional(),
  synopsis: z.string().optional(),
  cover: z.string().url().optional(),
  genre: z.string().optional(),
  year: z.coerce.number().int().optional(),
  status: BookStatusEnum,
  rating: z.coerce.number().int().min(1).max(5).optional(),
  notes: z.string().optional(),
})

export async function POST(request: NextRequest) {
  try {
    const body = await request.json()
    const result = CreateBookSchema.safeParse(body)
    if (!result.success) {
      return NextResponse.json(
        { error: 'Validation failed', details: result.error.flatten() },
        { status: 400 }
      )
    }
    const { slug } = await writeBook(result.data)
    return NextResponse.json({ slug }, { status: 201 })
  } catch (err) {
    console.error('[API] POST /api/books error:', err)
    return NextResponse.json(
      { error: 'Erro ao criar livro.' },
      { status: 500 }
    )
  }
}
```

Key details:
- `export const dynamic = 'force-dynamic'` prevents static pre-rendering (same pattern as page.tsx)
- Zod `CreateBookSchema` validates all fields with BookStatusEnum for status
- `z.string().url().optional()` for cover URL validation
- `z.coerce.number()` for year and rating (handles string-to-number from JSON)
- 201 Created on success with `{ slug }` in response body
- 400 with `.error.flatten()` on validation failure
- 500 with generic error message on unexpected errors
- Uses `[API]` log prefix for console.error (per PATTERNS.md log prefix convention)

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { z } from 'zod'
import { searchGoogleBooks } from '@/lib/api/google-books'
import { searchOpenLibrary } from '@/lib/api/open-library'

export const dynamic = 'force-dynamic'

const SearchSchema = z.object({
  query: z.string().min(2),
})

export async function POST(request: NextRequest) {
  try {
    const body = await request.json()
    const result = SearchSchema.safeParse(body)
    if (!result.success) {
      return NextResponse.json(
        { error: 'Validation failed', details: result.error.flatten() },
        { status: 400 }
      )
    }

    let results = await searchGoogleBooks(result.data.query)
    if (results.length === 0) {
      results = await searchOpenLibrary(result.data.query)
    }
    return NextResponse.json(results)
  } catch (err) {
    console.error('[API] POST /api/books/search error:', err)
    return NextResponse.json(
      { error: 'Erro ao buscar livros.' },
      { status: 500 }
    )
  }
}
```

Key details:
- Query must be at least 2 characters (prevents empty/single-char API calls)
- Google Books searched first; if 0 results, falls back to Open Library (per D-01 and CATALOG-01)
- Returns BookSearchResult[] directly (same shape from both providers)
- API keys stay server-side (never sent to client)

**Create `src/app/api/books/[slug]/route.ts`:**

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { z } from 'zod'
import { BookStatusEnum } from '@/lib/books/schema'
import { updateBook, deleteBook } from '@/lib/books/library-service'

export const dynamic = 'force-dynamic'

const UpdateBookSchema = z.object({
  title: z.string().min(1).optional(),
  author: z.string().min(1).optional(),
  isbn: z.string().optional(),
  synopsis: z.string().optional(),
  cover: z.string().url().optional(),
  genre: z.string().optional(),
  year: z.coerce.number().int().optional(),
  status: BookStatusEnum.optional(),
  rating: z.coerce.number().int().min(1).max(5).optional().nullable(),
  notes: z.string().optional(),
})

export async function PUT(
  request: NextRequest,
  { params }: { params: Promise<{ slug: string }> }
) {
  try {
    const { slug } = await params
    const body = await request.json()
    const result = UpdateBookSchema.safeParse(body)
    if (!result.success) {
      return NextResponse.json(
        { error: 'Validation failed', details: result.error.flatten() },
        { status: 400 }
      )
    }
    await updateBook(slug, result.data)
    return NextResponse.json({ ok: true })
  } catch (err) {
    console.error('[API] PUT /api/books/[slug] error:', err)
    if (err instanceof Error && err.message.includes('ENOENT')) {
      return NextResponse.json({ error: 'Livro nao encontrado.' }, { status: 404 })
    }
    return NextResponse.json({ error: 'Erro ao atualizar livro.' }, { status: 500 })
  }
}

export async function DELETE(
  _request: NextRequest,
  { params }: { params: Promise<{ slug: string }> }
) {
  try {
    const { slug } = await params
    await deleteBook(slug)
    return new NextResponse(null, { status: 204 })
  } catch (err) {
    console.error('[API] DELETE /api/books/[slug] error:', err)
    if (err instanceof Error && err.message.includes('ENOENT')) {
      return NextResponse.json({ error: 'Livro nao encontrado.' }, { status: 404 })
    }
    return NextResponse.json({ error: 'Erro ao excluir livro.' }, { status: 500 })
  }
}
```

CRITICAL details:
- `params` is a `Promise` in Next.js 15+ -- MUST use `await params` before accessing `slug` (per PATTERNS.md)
- PUT returns 200 with `{ ok: true }` on success
- DELETE returns 204 (no content) on success
- Both catch ENOENT errors and return 404 (file not found = book not found)
- `rating` in UpdateBookSchema is `.optional().nullable()` to allow clearing a rating
- `UpdateBookSchema` fields are all optional (partial update)
- Uses `[API]` log prefix in console.error

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client request -> API route | Untrusted JSON body from browser |
| API route -> filesystem | Validated data written to disk |
| API route -> external API | Search query forwarded to Google Books / Open Library |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Tampering | route.ts (invalid input) | mitigate | All inputs validated via Zod schemas with safeParse. Invalid data returns 400, never reaches filesystem. |
| T-02-11 | Tampering | [slug]/route.ts (path traversal via slug) | mitigate | Slug comes from URL param which is the filename. slug.ts generateSlug strips all non-alphanumeric chars. For update/delete, the slug is used as-is to find the file -- fs.readFile/unlink will fail with ENOENT if file doesn't exist, which returns 404. The slug cannot escape the library directory because getLibraryDir() joins the path. |
| T-02-12 | Information Disclosure | route.ts (error details) | mitigate | Zod validation errors use `.flatten()` (safe, no stack traces). Server errors return generic message, detailed error only in console.error. |
| T-02-13 | Denial of Service | search/route.ts | accept | External API calls are rate-limited by provider. Single-user app. Debounce in client UI (Plan 05) limits request frequency. |
